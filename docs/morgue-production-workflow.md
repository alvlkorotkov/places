# Morgue Production Workflow

Основа: `assets/images/Morgue/space-1.2.jpg`

## Что можно оставить Codex

- Зафиксировать структуру задачи в лендинге.
- Собирать prompt и negative prompt для каждого этапа.
- Формализовать clean plate, lighting pass и set dressing pass.
- Проверять визуал на соответствие исходнику: окна, дверь, ширина зала, перспектива пола.
- Готовить handoff для ретушера или художника.
- После ручной правки встраивать готовые изображения обратно в лендинг и публиковать их.

## Что лучше отдать человеку с Photoshop

- Ручное удаление люстр без потери потолочной лепнины.
- Ручное удаление растений, лавок и стола.
- Восстановление чистого пола и стен после удаления объектов.
- Сохранение всех 3 окон без изменения формы арок.
- Точная дорисовка двух люминесцентных ламп.
- Точная постановка 5 кушеток с телами под простынями, если нужна максимальная архитектурная точность.

## Рекомендуемый пайплайн

1. Сделать `clean plate` вручную поверх `space-1.2.jpg`, пока не убирая люстры.
2. Экспортировать этот этап как отдельный файл.
3. На clean plate зафиксировать более клиническое ощущение света, не ломая окна.
4. На следующем этапе добавить только `5` кушеток.
5. Проверить, что все `3` окна, дверь и ширина зала сохранены.
6. Уже после этого отдельно решить, нужно ли убирать люстры локальным ручным проходом.
7. Встроить результат в лендинг как финальную версию `Morgue`.

## Что проверять на каждом шаге

- Окон по-прежнему `3`.
- Центральная дверь остается на месте.
- Правый край кадра не схлопывается.
- Простенки между окнами не меняют ширину.
- Плитка пола не ломает перспективу.
- Потолок не теряет геометрию.
- Правая люстра не трогается глобальной генерацией, пока не собрана стабильная база.
